# UI Notes

## Visual direction
- clean, friendly, lightly game-adjacent
- modern utility app, not pixel-art fan site
- soft cards
- strong search
- tags and badges for explainability

## Interaction priorities
1. selecting Pokémon should feel fast
2. results should explain themselves
3. compare views should make differences obvious
4. users should never wonder what a score means

## Good UI patterns for this app
- sticky selection panel on desktop
- chips for filters
- section cards with headings + short helper text
- score badge + explainer accordion
- side-by-side compare rows
- saved team cards with compact summaries

## Home card copy ideas
- Item Optimizer: “Find the best items for a group.”
- Reverse Lookup: “Start from an item, habitat, or specialty.”
- Compatibility Explorer: “See who pairs best with a Pokémon.”
- Saved Teams: “Save and compare your best setups.”
