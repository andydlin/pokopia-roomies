# Acceptance Checklist

## Foundation
- [ ] App runs locally
- [ ] Routes exist
- [ ] JSON seed data loads successfully
- [ ] Types are centralized and reusable

## Item Optimizer
- [ ] User can select 1–5 Pokémon
- [ ] Shared favorite categories are shown
- [ ] Recommended categories are shown
- [ ] Recommended items are shown
- [ ] Coverage counts are visible
- [ ] Results update reactively

## Reverse Lookup
- [ ] User can filter by category
- [ ] User can filter by item
- [ ] User can filter by habitat trait
- [ ] User can filter by specialty
- [ ] Filters combine correctly
- [ ] Results explain why each Pokémon matched

## Compatibility Explorer
- [ ] Pokémon detail page renders
- [ ] Best teammate candidates are ranked
- [ ] Scores are visible
- [ ] Score breakdown is understandable
- [ ] Habitat conflicts are surfaced

## Teams + Compare
- [ ] User can save a team
- [ ] Team requires 2–5 Pokémon
- [ ] localStorage persistence works
- [ ] Team detail page works
- [ ] Compare view works side-by-side

## Quality
- [ ] Domain logic is not duplicated across pages
- [ ] No giant monolithic components
- [ ] No hardcoded UI-only hacks for scoring
