# Acceptance Checklist

- [ ] Home Builder is the primary product surface.
- [ ] No user-facing numeric compatibility scores.
- [ ] Users can add any pokemon/item/habitat regardless of fit.
- [ ] Sticky home summary is always accessible on desktop.
- [ ] Mobile builder is accessible via bottom bar/sheet.
- [ ] Saved homes can be created, loaded, duplicated, renamed, deleted.
- [ ] Current home persists locally.
- [ ] Restore-code flow exists in architecture and implementation/contracts.
- [ ] Item browser supports contextual + all modes.
- [ ] Neutral items remain addable and visible.
- [ ] Navigation preserves browse context.
- [ ] Expanded home view provides jump actions into contextual browsers.
- [ ] Pokedex exists as separate neutral browse surface.
- [ ] Legacy dead code/tool-first wording removed or deprecated.
